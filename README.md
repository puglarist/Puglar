## üß† Unreal/Unity-Style Architecture

### 1) Scene-Based Training Design
Treat each session as a **Scene/Level**:
- **Exploration Scene** ‚Üí awareness + environmental reads.
- **Movement Scene** ‚Üí body mechanics + positional transitions.
- **Command Scene** ‚Üí stances + decision cadence.
- **Endurance Scene** ‚Üí progressive load and recovery.

### 2) Component-Driven Skill Building
Treat every skill as a **Component** attached to the trainee:
- Perception component
- Posture component
- Locomotion component
- Breath/recovery component
- Command/timing component

Swap components in or out per day, just like toggling gameplay components.

### 3) Gameplay Loop (Fixed Update)
Use a stable runtime loop:
- **Init** (2‚Äì5 min): alignment, breath, intent.
- **Tick Block A** (10‚Äì20 min): primary drill.
- **Tick Block B** (10‚Äì20 min): support drill.
- **Challenge Pass** (5‚Äì10 min): pressure/variation.
- **Cooldown + Debrief** (5 min): logging and calibration.

### 4) Difficulty Curves (Progression System)
Scale like game difficulty tiers:
- **Tier 1**: static, precise, low fatigue.
- **Tier 2**: dynamic transitions, moderate fatigue.
- **Tier 3**: mixed conditions, cognitive load, time pressure.

### 5) Build Targets (Platform Thinking)
Choose the format by context:
- **Desktop build** = full handbook study.
- **Mobile build** = pocket cards.
- **Field build** = waterproof / laminated cards.
